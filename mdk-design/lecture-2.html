Лекция 2: Разработка технического задания, эскизного и технического проекта. Этапы проектирования ИС

1. Разработка технического задания (ТЗ)

Техническое задание - это документ, содержащий технические требования к разрабатываемой информационной системе и определяющий порядок ее создания.

Содержание ТЗ:

  • Общие сведения о системе
  • Назначение и цели создания системы
  • Требования к системе
  • Состав и содержание работ по созданию системы
  • Порядок контроля и приемки системы
  • Требования к документации
  • Этапы и сроки разработки

2. Эскизный проект

Эскизный проект - это стадия проектирования, на которой определяются основные архитектурные решения системы.

Результаты эскизного проектирования:

  • Архитектура системы
  • Основные функциональные модули
  • Схемы взаимодействия компонентов
  • Предварительные спецификации интерфейсов
  • Оценка необходимых ресурсов
  • План дальнейших работ

3. Технический проект

Технический проект - это детальная разработка всех компонентов системы на основе утвержденного эскизного проекта.

Компоненты технического проекта:

  • Детальные спецификации всех модулей
  • Схемы баз данных
  • Интерфейсы пользователя
  • Алгоритмы обработки данных
  • Протоколы взаимодействия
  • Требования к аппаратному обеспечению
  • План тестирования

4. Этапы проектирования ИС

  1. Предпроектное обследование

    Анализ предметной области, сбор требований, изучение существующих систем

  2. Разработка технического задания

    Формирование требований, определение целей и задач системы

  3. Эскизное проектирование

    Разработка архитектуры системы, выбор технологий, создание прототипов

  4. Техническое проектирование

    Детальная разработка всех компонентов системы

  5. Реализация

    Программирование, настройка, интеграция компонентов

  6. Тестирование

    Проверка функциональности, производительности, безопасности

  7. Ввод в эксплуатацию

    Установка системы, обучение пользователей, переход на новую систему

  8. Сопровождение и развитие

    Поддержка системы, устранение ошибок, доработки и модернизация

5. Методологии проектирования

  • Waterfall (каскадная модель) - последовательное выполнение этапов
  • Agile (гибкая методология) - итеративная разработка
  • Scrum - фреймворк для Agile-разработки
  • RAD (быстрая разработка приложений) - прототипирование и быстрая разработка
  • Spiral model (спиральная модель) - итеративная разработка с оценкой рисков
