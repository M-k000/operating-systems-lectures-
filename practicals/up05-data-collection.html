УП.05: Сбор исходных данных для разработки проектной документации на информационную систему

Темы практической работы:

  • Тема 1: Основные понятия и определения

    Понятие информационной системы, виды ИС, жизненный цикл

  • Тема 2: Методы сбора данных

    Интервью, анкетирование, наблюдение, анализ документов

  • Тема 3: Требования к исходным данным

    Функциональные, нефункциональные, бизнес-требования

  • Тема 4: Документирование требований

    SRS, use cases, пользовательские истории, спецификации

  • Тема 5: Анализ и валидация данных

    Проверка полноты, непротиворечивости, актуальности данных

  • Тема 6: Инструменты и технологии

    Средства моделирования, системы управления требованиями

Тема 1: Основные понятия и определения

Информационная система (ИС) - это система, предназначенная для хранения, поиска и обработки информации, а также соответствующие организационные ресурсы, которые обеспечивают и распространяют информацию.

Виды информационных систем:

  • Транзакционные системы (OLTP)
  • Системы поддержки принятия решений (DSS)
  • Информационно-справочные системы
  • Экспертные системы
  • Системы управления предприятием (ERP)

Жизненный цикл информационной системы:

  1. Анализ требований
  2. Проектирование
  3. Разработка
  4. Тестирование
  5. Внедрение
  6. Эксплуатация и сопровождение

Тема 2: Методы сбора данных

Интервью

Структурированная беседа с заинтересованными лицами для выявления требований и потребностей.

Анкетирование

Массовый сбор информации через стандартизированные опросники.

Наблюдение

Изучение рабочих процессов в естественных условиях.

Анализ документов

Изучение существующей документации и регламентов.

Тема 3: Требования к исходным данным

Функциональные требования

Определяют, что система должна делать: функции, операции, данные.

Нефункциональные требования

Определяют, как система должна работать: производительность, надежность, безопасность.

Бизнес-требования

Определяют цели бизнеса, которые должны быть достигнуты с помощью системы.

Тема 4: Документирование требований

Основные документы:

  • SRS (Software Requirements Specification) - спецификация требований к программному обеспечению
  • Use Cases - сценарии использования системы
  • User Stories - пользовательские истории в формате "Как [роль], я хочу [функция], чтобы [ценность]"
  • Технические спецификации - детальное описание архитектуры и компонентов

Тема 5: Анализ и валидация данных

Критерии качества исходных данных:

  • Полнота - все необходимые данные собраны
  • Непротиворечивость - отсутствие противоречий между требованиями
  • Актуальность - данные соответствуют текущему состоянию бизнес-процессов
  • Проверяемость - возможность объективной проверки выполнения требований
  • Однозначность - четкое и недвусмысленное толкование

Тема 6: Инструменты и технологии

Средства моделирования:

  • UML-диаграммы (Use Case, Sequence, Class, Activity)
  • BPMN-нотация для бизнес-процессов
  • ER-диаграммы для моделирования данных

Системы управления требованиями:

  • JIRA
  • Confluence
  • IBM DOORS
  • Modern Requirements
